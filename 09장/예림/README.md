# 09장. 문서 품질 측정하기

## 📌 1. 문서 품질 이해하기
- 문서 품질을 측정하기 전에 '품질'을 정의하자.
  - 구글의 연구 그룹은 문서 품질에 대해 다음과 같이 간단하게 정의했다.
    > 목적을 달성하는 문서가 좋은 문서다.
- 목적을 달성하는 문서가 좋은 문서라면, 문서의 목적은 무엇인가?
- 문서의 목적은 **코드의 목적과 방향이 일치해야 한다.**
- 구글의 연구 그룹은 문서 품질을 두 가지 기본 범주로 분류했다.
  - 기능적 품질 : 문서가 목적이나 목표를 달성하는지를 의미
  - 구조적 품질 : 문서가 잘 작성되고 잘 구조화되었는지
### 기능적 품질
기능적 품질은 문서가 이루고자 하는 목표를 달성하는지 나타낸다. 문서의 기능적 품질은 다음 범주로 나눌 수 있다.
- 목적성
- 검색성
- 정확성
- 완정성
#### 접근성
- 기능적 품질에서 가장 필수적인 측면
- 접근성을 판단하는 요소로는 언어, 읽기 수준, 스크린 리더 액세스가 있음
  - 언어 : 독자가 사용하는 언어로 작성하는 것이 중요함
  - 읽기 수준 : 기술 문서는 모든 부분이 고등학교 1~2학년 수준으로 작성되어야 함
    - 문서의 읽기 수준을 측정하는 방법으로는 플래시-킨테이드 등급 수준, 자동화된 가독성 지수, 콜먼-리아우 지수 등이 있음
    - 어떤 사람이 글을 이해하는 데 필요한 최저 학년 수준을 측정해줌
  - 스크린 리더 액세스 : 문서에 사용하는 모든 시각적 요소에 대체 텍스트 사용, 링크하는 모든 비디오에는 자막이 있어야 함
#### 목적성
- 문서의 제목과 첫 번째 단락에 문서의 목적과 독자가 무엇을 달성하는 데 도움을 주는지 명시해야 함
#### 검색성
- 독자가 얼마나 쉽게 콘텐츠를 찾아내고 콘텐츠 내부에서 정보를 탐색하는지를 나타내는 척도
- 사용자가 무엇을 검색하고자 하는지 이해하고, 검색 키워드를 표준화하며, 사용자가 사이트를 어떻게 찾아서 들어오는지 모니터링하는 일이 검색성을 높이는 데 도움이 됨
- 콘텐츠 전체에서 바라본 독자의 현재 위치에 대해 가능한 한 많은 맥락을 제공해야 함
#### 정확성
- 정확성이 높은 문서에는 잘 작동하는 샘플 코드 및 명령줄 예제와 더불어, 설명하는 코드에 대해 정확하면서도 최신 정보가 잘 반영된 기술적 설명이 나옴
#### 완전성
- 독자가 목표를 성공적으로 달성하는 데 필요한 모든 정보를 포함해야 함
- 완전한 문서가 되려면 다음과 같아야 함
  1. 독자가 따라야 하는 모든 필수 조건을 나열한다
  2. 해당 작업을 완료하는 데 필요한 모든 작업을 문서화한다
  3. 독자가 취해야 할 다음 단계를 정의한다

### 구조적 품질
문서가 얼마나 잘 작성되었는지를 나타낸다. 문장, 단락, 제목 구조, 언어 품질, 문법 정확도가 포함된다.
이 책에서는 좋은 글의 속성으로 알려진 '세가지 C'를 사용하여 구조적 품질을 정의한다.
- 명확성(clarity)
- 간결성(conciseness)
- 일관성(consistency)

#### 명확성
- 문서가 얼마나 이해하기 쉬운지를 나타내는 척도
- 명확한 문서는 다음과 같은 요소를 포함해야 한다
  - 문서 주제를 논리적 단위로 나누며, 잘 정의되고 적절한 순서로 배치된 섹션 제목
  - 작업 및 각 단계에 대해 시간순으로 구성되며, 각각의 기대 결과를 나타내는 섹션 제목
  - 프로세스의 각 단계에 대한 명확한 결과
  - 독자가 이해할 수 있는 방식으로 구성된 단계
  - 독자가 막힐 가능성이 있는 모든 위치를 미리 알려주는 콘텐츠
  - 사용자가 겪을 수 있는 모든 오류의 정의
#### 간결성
- 간결성에 대한 좋은 정의는 '간단하지만 포괄적'이라는 것이다
- 간결한 문서란?
  - 문서 전체 수준 : 독자 및 독자의 목표와 관련된 정보만 포함된 문서
  - 문장 및 단어 수준 : 독자에게 필요한 정보만 있고 그 이상의 내용은 없어야 한다. 불필요한 단어와 개념 포함 X)
#### 일관성
- 콘텐츠의 구조, 소개하는 개념, 단어 선택 등이 문서 전체에서 일관됨
- 일관성 있는 문서란?
  - 문서 전체 수준 : 문서 제목, 섹션 제목, 단락 구조, 목록에 일관성이 있어야 한다
  - 문장 수준 : 어떤 개념을 나타낼 떄 같은 용어나 표현을 사용한다
- 스타일 가이드와 표준화된 문서 유형을 사용하면 콘텐츠의 일관성을 높이는 데 도움이 된다

### 기능적 품질과 구조적 품질의 연관성
- 기능적 품질이 더 중요하다
- 기능적 품질 낮음 + 구조적 품질 높음 = 전체 품질 좋지 않음
- 기능적 품질 높음 + 구조적 품질 양호함 = 전체 품질 좋음

## 📌 분석을 위한 전략 세우기
- 문서가 효과적이려면 기술 목표와 비즈니스 목표의 방향을 독자의 목표와 방향에 맞춰야 한다
- 효과적인 분석 전략을 수집하려면 다음을 명확하게 정의해야 한다
  - 조직의 목표와 측정 방법
  - 독자의 목표와 측정 방법
  - 문서의 목표와 측정 방법
### 조직의 목표와 메트릭
- 조직의 목표는 일반적으로 수익과 관련이 있음
### 사용자의 목표와 메트릭
- 특정 작업을 완료하는 데 초점이 맞춰짐
### 문서의 목표와 메트릭
- 수집할 수 있는 문서 메트릭은 다음과 같다
  - 순 방문자 수
  - 페이지 조회수
  - 페이지에 머문 시간
  - 이탈률
  - 검색 키워드 분석
  - 읽기 수준 분석
  - 문서와 관련된 고객 지원 문제
  - 링크 유효성 검사
  - 타임 투 헬로월드
## 📌 문서 메트릭 사용 팁
### 계획 세우기
- 답하고자 하는 구체적인 질문을 만들어라.
- 최소한 다음 질문에는 답해야 한다
  - 이 항목을 측정하려는 이유
  - 그 정보로 무엇을 할 것인가?
  - 이 노력이 조직의 목표 달성에 어떻게 도움이 되는가?
### 기준선 설정하기
- 변경 전후의 메트릭을 비교하여 변경 작업의 영향 평가 가능
### 맥락 고려하기
- 독자는 자신의 필요에 따라 다양한 방식으로 문서 사용
- 이는 맥락에 따라 의미가 달라지는 메트릭에서 잘 드러남
### 여러가지 메트릭 함께 사용하기
- 여러 메트릭 간의 연관관계를 도출할 수 있다면 더 나은 결과를 얻을 수 있음
### 정성적 피드백과 정량적 피드백 섞기
- 정성적 피드백과 정량적 피드백을 모두 살펴보는 것이 중요
